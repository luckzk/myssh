# 03 总体架构

## 3.1 分层架构

复制版采用经典四层 + 旁路审计的结构：

```
┌─────────────────────────────────────────────────────────┐
│  接入层  Nginx / Caddy（TLS、静态资源、WebSocket 反代）   │
└───────────────┬─────────────────────────────────────────┘
                │
┌───────────────▼─────────────────────────────────────────┐
│  前端 SPA  React + AntD + xterm.js + Monaco（PWA）        │
└───────────────┬─────────────────────────────────────────┘
                │ REST (/api/*)  +  WebSocket (/ws/*)
┌───────────────▼─────────────────────────────────────────┐
│  应用层  Go 服务（单体可拆分）                            │
│  ├─ HTTP API（账号/资产/授权/日志…）                      │
│  ├─ 会话网关（SSH 原生 / guacd 代理 / DB 代理）           │
│  ├─ 审计管道（命令、文件、录像、SQL）                     │
│  └─ 调度器（计划任务、监控采集、会话清理）                │
└───────┬───────────────┬───────────────┬─────────────────┘
        │               │               │
   ┌────▼────┐     ┌────▼────┐     ┌────▼─────┐
   │ 数据库  │     │  guacd  │     │ 对象存储 │
   │ PG/MySQL│     │RDP/VNC/ │     │ 录像/文件│
   │ /SQLite │     │ Telnet  │     │ 本地/S3  │
   └─────────┘     └────┬────┘     └──────────┘
                        │
                ┌───────▼────────┐
                │  被管资产/目标  │
                │ 主机 / 数据库   │
                └────────────────┘
```

## 3.2 核心组件职责

| 组件 | 职责 | 关键点 |
| --- | --- | --- |
| **API 服务** | 业务 CRUD、认证鉴权、菜单下发 | 统一 `{code,message,data}` 包络；不透明令牌 + 服务端会话表 |
| **会话网关** | 建立到目标的连接并双向转发 | SSH 用 Go 原生 `crypto/ssh`；RDP/VNC/Telnet 经 guacd |
| **审计管道** | 旁路截获并持久化所有交互 | 终端流写录像；命令解析写日志；文件操作留痕 |
| **调度器** | 周期任务 | 计划任务、资产探活、过期会话回收、录像归档 |
| **guacd** | 图形协议转换 | 独立进程/容器，只跟会话网关用 guacamole 协议通信 |
| **存储** | 录像、上传文件、证书 | 抽象 Storage 接口，支持本地盘与 S3 兼容 |

## 3.3 一次 SSH 会话的数据流（核心闭环）

```
用户点击"连接资产"
  → 前端开 WebSocket /ws/ssh?assetId=...&token=...
  → 网关校验令牌 + 鉴权(该用户对该资产是否授权)
  → 网关取出资产绑定的凭证(解密) → crypto/ssh 拨号到目标
  → 建立 PTY，双向桥接 (xterm.js ⇄ WebSocket ⇄ ssh channel)
  ┌─ 旁路 ─────────────────────────────────────────┐
  │ ① 终端输出流 → 写入录像文件(asciinema/自定义帧)   │
  │ ② 输入行缓冲 → 命令解析 → exec-command-log        │
  │ ③ 命中 command-filter → 阻断/告警/记录            │
  │ ④ SFTP 操作 → filesystem-log + storage           │
  └──────────────────────────────────────────────────┘
  → 会话结束 → 落 offline-session 记录 + 录像地址
```

> 图形协议（RDP/VNC）数据流类似，区别是**终端桥接换成 guacd**：前端用 guacamole-common-js 渲染，网关把浏览器与 guacd 之间的 guacamole 指令流转发，并旁路录制为 guacd 的录像格式。详见 [08 协议接入](/plan/08-protocols)。

## 3.4 认证与会话模型

基于 [02 探查](/plan/02-recon) 的发现：

- **令牌**：登录返回 `NT_`-style 不透明令牌，存服务端会话表（含 userId、过期、来源 IP、是否 MFA 通过）。
- **双通道携带**：`X-Auth-Token` 头 或 HttpOnly Cookie。
- **强制下线**：因为是服务端会话表（非纯 JWT），管理员可在「在线会话」一键失效。
- **MFA**：登录返回 `needTotp`，二阶段校验 TOTP。
- **菜单鉴权**：登录后 `/api/account/info` 按角色计算 `menus[].checked` 下发，前端动态渲染。

## 3.5 部署形态（先单体，后可拆）

- **MVP/中小规模**：单 Go 二进制 + guacd + 数据库 + Nginx，docker-compose 一把起。
- **规模化**：会话网关水平扩展（多实例），前面加 LB；guacd 多实例；录像入对象存储；数据库主从。
- 详见 [11 部署与运维](/plan/11-deployment)。
