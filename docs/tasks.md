# 任务清单

## K · OxideTerm 借鉴能力落地

目标：吸收 OxideTerm 中适合 Web 堡垒机的高价值能力，保持本项目“受控访问 + 审计留痕”的定位。

| 编号 | 任务 | 状态 | 交付口径 |
| --- | --- | --- | --- |
| K1 | 端口转发管理与审计 | 完成 | 支持 SSH 会话上的本地转发、远程转发、动态 SOCKS5 转发，能启动/停止/查看状态，并记录操作者、资产、端口、目标和退出原因 |
| K2 | SSH HostKey TOFU | 完成 | 首次连接记录主机指纹；后续指纹一致放行，变更则阻断并要求管理员确认；保留 `known_hosts` 生产收紧模式 |
| K3 | 断线宽限期重连 | 完成 | 终端 WS 异常断开后进入宽限期，前端短时间内用同一 `sessionId` 重连并延续审计记录；恢复失败则结束旧会话并允许新建 |
| K4 | SFTP 队列与预览 MVP | 完成 | 文件管理支持多文件上传队列、进度/失败提示、文本/图片预览；传输和预览相关操作继续走文件审计 |

## 约束

- 不复制 OxideTerm GPL-3.0 代码，只借鉴产品和架构思路。
- 所有新能力必须保留服务端鉴权和审计，不把凭证明文下发前端。
- 高风险动作（转发、批量传输、HostKey 变更确认）需要可追踪到用户与资产。

## 已知边界

- K3 当前是“同会话审计续接 + 自动重拨 SSH”的宽限重连，不是 PTY 进程级保活；网络断开期间运行中的 vim/htop 等远端 TUI 进程仍可能随原 SSH 断开而结束。
- K4 预览限制单文件 2MB，优先覆盖文本和常见图片，复杂二进制格式仍建议下载查看。
